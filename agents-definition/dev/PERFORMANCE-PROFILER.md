# AGENT_SPEC: Performance-Profiler

## 에이전트 정보
- **이름**: Performance-Profiler
- **분류**: 개발 에이전트 (dev)
- **목표**: 성능 병목 자동 감지 및 최적화 제안
- **우선순위**: 🔴 Phase 1 (2026-05-14 배포 목표)
- **상태**: 설계 중

---

## 담당 업무

### 입력 (Input)
```
1. 벤치마크 데이터
   - 실행 시간 (ms)
   - 메모리 사용량 (MB)
   - CPU 사용률 (%)
   - 함수별 호출 시간

2. 프로파일링 결과
   - Flamegraph 데이터
   - 함수 호출 그래프
   - 메모리 할당 추적

3. 비교 기준
   - 이전 버전 성능
   - 성능 목표 (SLA)
   - 회귀 감지 기준값
```

### 처리 로직
```
Step 1: 데이터 수집 (구조)
  - 벤치마크 자동 실행
  - 프로파일링 도구 실행 (FreeLang, Go)
  - 메트릭 정규화

Step 2: 핫스팟 감지 (AI + 구조)
  - 상위 5개 느린 함수 식별
  - 병목 원인 분류:
    * 알고리즘 비효율 (시간복잡도)
    * 메모리 할당 (GC pressure)
    * I/O 대기 (락, DB 쿼리)
    * 불필요한 작업

Step 3: 최적화 제안 (AI)
  - 알고리즘 개선 코드 생성
  - 캐싱 전략 제안
  - 병렬화 가능성 분석

Step 4: 검증 (구조)
  - 최적화 코드 벤치마크 실행
  - before/after 비교
  - 정확도 검증 (결과 동일성 확인)
  - 회귀 테스트

Step 5: 리포트 생성 (구조)
  - 개선율 시각화
  - 상세 분석 리포트
```

### 출력 (Output)
```
1. 성능 분석 리포트
   - 핫스팟 순위 (상위 5개)
   - 병목 원인 분류
   - 개선 가능성 평가

2. 최적화 코드
   - 제안된 수정 코드 (최적화 버전)
   - 임팩트 분석 (예상 개선율)
   - 부작용 평가 (메모리 증가, 복잡도 등)

3. 벤치마크 결과
   - before/after 비교 (시간, 메모리, CPU)
   - 성능 개선율 (%)
   - 회귀 감지 (SLA 기준)

4. 메트릭 대시보드
   - 성능 추이 그래프 (시간 축)
   - 각 모듈별 성능 비교
   - 알람 설정 (임계값)
```

## 성공 기준

| 기준 | 목표 | 검증 방법 |
|------|------|---------|
| **핫스팟 감지 정확도** | 상위 5개 90%+ 정확 | 프로파일링 도구와 비교 |
| **최적화 제안 품질** | 제안된 코드 80%+ 적용 가능 | 수동 검증 + 테스트 |
| **개선율** | 평균 30% 이상 성능 개선 | 벤치마크 재실행 |
| **정확도 보장** | 최적화 코드 결과 100% 동일 | 자동 테스트 |
| **처리 시간** | 1시간 벤치마크 분석 < 5분 | 성능 테스트 |

## 에이전트 한계점 & 보완 구조

### AI 한계점
- ❌ 복잡한 알고리즘 최적화 자동 생성 어려움 (도메인 지식 필요)
- ❌ 마이크로최적화(Micro-optimization) 판단 어려움
- ❌ 실제 프로파일링 데이터 직접 수집 불가

### 보완 구조 (도구 + AI)
1. **데이터 수집**: perf, flamegraph 등 도구 실행 (구조)
2. **분석**: AI가 병목 원인 분류 (AI)
3. **최적화 코드**: AI가 개선 코드 생성 (AI)
4. **검증**: 자동 벤치마크 + 테스트 (구조)
