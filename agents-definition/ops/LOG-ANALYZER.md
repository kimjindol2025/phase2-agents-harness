# AGENT_SPEC: Log-Analyzer

## 에이전트 정보
- **이름**: Log-Analyzer
- **분류**: 운영 에이전트 (ops)
- **목표**: 에러 로그 자동 분류, 경향 분석, 근본 원인 추출
- **우선순위**: 🔴 Phase 1 (2026-05-07 배포 목표)
- **상태**: 설계 중

---

## 담당 업무

### 입력 (Input)
```
1. 애플리케이션 로그
   - FreeLang 컴파일러 에러
   - gogs-server-fl HTTP 에러
   - BigWash 서비스 에러
   - 시스템 로그

2. 로그 포맷
   - 타임스탬프, 레벨(ERROR/WARN/INFO), 메시지
   - 스택 트레이스
   - 컨텍스트 정보 (사용자, 요청ID)
```

### 처리 로직
```
Step 1: 로그 파싱 (구조)
  - 타임스탬프 추출
  - 에러 레벨 분류
  - 메시지 정규화

Step 2: 자동 분류 (AI)
  - 에러 타입 판단 (DB 에러, 네트워크, 권한, 타입 불일치 등)
  - 심각도 평가 (Critical/High/Medium/Low)
  - 영향받는 서비스 식별

Step 3: 경향 분석 (구조)
  - 시간대별 에러 분포 (히스토그램)
  - 에러 빈도 추이 (증가/감소)
  - 에러 그룹화 (같은 유형 에러 클러스터링)

Step 4: 근본 원인 추출 (AI)
  - 스택 트레이스 분석
  - 연관 에러 추적 (cascade failure)
  - 가설 제시 ("DB 연결 풀 고갈" 등)

Step 5: 대시보드 생성 (구조)
  - 일일/주간 리포트
  - 심각도별 요약
  - 추천 조치 사항
```

### 출력 (Output)
```
1. 에러 분류 리포트 (Markdown)
   - 발생 에러 요약 (타입별, 시간별)
   - 심각도 분포 (pie chart)
   - 상위 10개 에러

2. 경향 분석
   - 시간대별 에러율 그래프
   - 서비스별 에러 비교
   - 최근 변화 추세

3. 근본 원인 분석
   - "Probable Root Cause: DB Connection Pool Exhausted"
   - 관련 스택 트레이스 (top 3)
   - 권장 해결책

4. 자동 생성 메타데이터
   - 처음 발생 시점
   - 마지막 발생 시점
   - 발생 횟수 & 추이
```

---

## 성공 기준

| 기준 | 목표 | 검증 방법 |
|------|------|---------|
| **분류 정확도** | 에러 타입 90%+ 정확 | 수동 샘플 검증 |
| **심각도 판단** | Critical 오판 제로 | 운영팀 재확인 |
| **근본원인 적중율** | 60%+ (정확한 근본원인) | 실제 패치 결과 비교 |
| **리포트 완성도** | 주요 에러 100% 포함 | 운영팀 만족도 |
| **처리 시간** | 1GB 로그 분석 < 1분 | 성능 테스트 |

## 에이전트 한계점 & 보완 구조

### AI 한계점
- ❌ 매우 큰 로그 파일 일괄 분석 불가 (토큰 제한)
- ❌ 도메인별 커스텀 에러 판단 기준 학습 불가

### 보완 구조 (스트리밍 + AI)
1. **로그 스트리밍**: 로그를 배치로 나누어 처리 (구조)
2. **필터링**: 에러만 추출 (구조)
3. **분류**: AI가 에러 타입 판단 (AI)
4. **통합**: 배치별 결과 병합 (구조)
